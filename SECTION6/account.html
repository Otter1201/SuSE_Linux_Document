    帳號管理

    在SuSE linux下，使用者與群組帳號密碼等資訊被記錄在以下幾個檔案中:

    /etc/passwd

    root:~ # cat /etc/passwd # 記錄所有帳號資料
    root:x:0:0:root:/root:/bin/bash
    bin:x:1:1:bin:/bin:/bin/bash
    {account}:1025:1000:{info}:/home/account:/bin/bash
    ...

    各個欄位說明如下:

    1 {account} 帳號名稱
    2 x # password
    此欄位資料已經被移至/etc/shadow中，所以僅顯示x
    3 1025 # UID
    root UID=0
    system daemon UID=1~999
    user UID=1000~
    4 1000 # GID
    記錄在 /etc/group
    5 {info} 使用者資訊說明欄，或用finger來取得更詳細的資訊
    6 /home/account 使用者的家目錄
    root的家目錄位於/root底下
    7 /bin/bash Shell

    /etc/shadow

    root:~ # cat /etc/shadow # 記錄密碼資料
    {account}:$*****:16001:0:99999:7::17075:

    各個欄位說明如下:

    1 {account} 帳號名稱
    2 $***** # passwd
    這個欄位內的資料才是真正的密碼，而且是經過編碼的密碼(加密)
    3 16001 這個欄位記錄了『更動密碼那一天』的日期，自1970/01/01算起之天數，欲知道正確之日期可用後述chage -l指令
    4 0 密碼不可被更動的天數
    5 99999 密碼需要重新變更的天數，99999表示密碼變更沒有強制性
    6 7 密碼需要變更期限前的警告天數
    7 (space) 密碼過期後的帳號寬限時間(密碼失效日):密碼有效日期為『更新日期(第3欄位)』+『重新變更日期(第5欄位)』，過了該期限後使用者依舊沒有更新密碼，那該密碼就算過期了。 雖然密碼過期但是該帳號還是可以用來進行其他工作的，包括登入系統取得 bash。不過如果密碼過期了，那當你登入系統時，系統會強制要求你必須要重新設定密碼才能登入繼續使用喔，這就是密碼過期特性。
    那這個欄位的功能是什麼呢？是在密碼過期幾天後，如果使用者還是沒有登入更改密碼，那麼這個帳號的密碼將會『失效』，亦即該帳號再也無法使用該密碼登入了。要注意密碼過期與密碼失效並不相同。
    8 17075 帳號失效日期(到期日)，一樣是以1970/01/01為第一天算起
    9 (space) 最後一個欄位是保留的，看以後有沒有新功能加入。

    /etc/group

    root:~ # cat /etc/group # 查看群組資訊
    bin:x:1:daemon
    daemon:x:2:
    group:!:1000:{account}

    各個欄位說明如下:

    1 group 群組名稱
    2 x 群組密碼，通常不需要設定，此欄位資料已經被移至/etc/gshadow中，所以僅顯示x
    3 1000 # GID
    4 {account} 加入此群組的帳號

    新增使用者 (useradd)
    root:~ # useradd {account} # 新增一個名為{account}的使用者
    root:~ # passwd {account} # 設定account登入密碼
    Changing password for user1.
    New Password :
    Reenter New Password :
    Password changed. # 密碼設定無誤
    root:~ # finger {account} # 查詢該帳號使用者資訊
    root:~ # su - {account} # 以account身分登入
    su: warning: cannot change directory to /home/account: No such file or directory # useradd並沒有為account建立家目錄
    account@linux: /root> # 故登入時進入/root目錄下

    arguments

    argument description example
    -m, --creat-home # Create home directory
    建立帳號同時建立家目錄    		 useradd -m {account}
    -e, --expire YYYY-MM-DD # Expire date
    設定帳號到期日:年-月-日    		 useradd -e 2007-12-01 {account}
    -s, --shell SHELL # Login shell: /bin/sh, /bin/csh, /bin/tcsh, /bin/zch, /bin/false
    設定登入的shell, false表示該使用者不可登入    		 useradd -s /bin/sh {account}
    -u, --uid UID # User id
    使用者的UID, root UID=0, 0~1000為系統服務，在1000以後為一般使用者    		 useradd -u 1007 {account}
    -g, --gid GID # Main group
    設定使用者的主要群組，可為數字或是群組名稱    		 useradd -g {group} {account}
    -G, --groups group # Subgroup
    設定使用者的次要群組，當有多個次要群組需加入時，可用","分隔    		 useradd -G {group1},{group2} {account}
    -d, --home HOMEDIR # Home directory
    設定使用者的家目錄    		 useradd -h /home/account {account}
    -k, --skel SKELDIR # Skel directory
    建立使用者家目錄後，要從哪個目錄底下複製預設設定檔    		 useradd -k /etc/skel {account}
    -c, --comment COMMENT # User finger information (Name:)
    設定使用者的名稱，可用finger指令查詢    		 useradd -c {name} {account}

    在新增使用者時，系統會有預設的設定檔來設定一些參數，在/etc/default/useradd

    root:~ # cat /etc/default/useradd
    GROUP=100 # 設定帳號的主要群組
    HOME=/home # 設定帳號的家目錄
    INACTIVE=-1 # 設定帳號密碼到期後，幾天後會被鎖（即是不可以登入系統)
    EXPIRE= # 設定帳號的到期日
    SHELL=/bin/bash # 設定帳號的登入 SHELL
    SKEL=/etc/skel # 設定帳號的 SKELDIR
    GROUPS=dialout,uucp,video,audio # 設定帳號的次要群組
    CREATE_MAIL_SPOOL=no #不建立使用者的mailbox

    帳號修改 (usermod)與刪除 (userdel)

    帳號建立後，如果需要修改其設定可用usermod指令，參數和上述用法均相同

    root:~ # usermod -[e,s,u,g,G,d] {account}

    移除帳號時，使用usrdel指令

    root:~ # userdel {account} # 刪除該帳號，但不會刪除其家目錄

    arguments

    argument description example
    -r, --rmove-home # Remove home directory & mail spool
    在移除帳號同時也一起移除使用者家目錄和 mail    		 userdel -r {account}

    密碼設定 (passwd)

    在使用者密碼的部分，使用 passwd這支指令

    root:~ # passwd {account} # Change password for user
    # root可以變更所有user和自己的密碼，但user使用這支指令只能變更自己的密碼
    root:~ # passwd -S {account} # 查看密碼狀態
    {account} PS MM/DD/YYYY 0 99999 7 -1

    各個欄位說明如下:

    column description
    {account} 帳號
    PS/LK/NP 表示密碼狀態
    PS 為有密碼
    LK 為鎖住密碼
    NP 為無密碼
    MM/DD/YYYY 表示密碼最後修改日期
    0 使用者至少隔幾天才能改密碼
    99999 使用者密碼還有幾天會無效
    7 警告密碼到期的期限，預設是 7 天，表示密碼到期的前 7 天，只要使用者登入系統，系統都會警告
    -1 預設是 "-1" ，功能如下面 -i參數所述

    arguments

    argument description example
    -n MIN # minimum number of days between password changes
    使用者至少幾天隔幾天才能改密碼    		 passwd -n 14 {account}
    -x MAX # maximum number of days during which a password is valid
    使用者密碼還有幾天會無效    		 passwd -x 30 {account}
    -w WARN # number of days of warning before a password change
    設定當密碼有效期限幾天內，系統會警告使用者密碼無效    		 passwd -w 7 {account}
    -i INACT # number of days of inactivity after a password has expired before the account is locked
    在使用者密碼到期後，
    "-1":當使用者下次登入將被強迫更改密碼
    "0":取消這功能，即是沒有什麼事情發生    		 passwd -i -1 {account}
    -l # lock password, let user cannot login
    鎖住使用者密碼，讓使用者無法登入    		 passwd -l {account}
    -u # unlock passord, user can login
    解開使用者密碼，讓使用者可以登入    		 passwd -u {account}
    -S # password status
    查看使用者密碼狀態    		 passwd -S {account}
    -a # all accounts
    查看所有使用者密碼狀態，與-S 一起使用    		 passwd -Sa {account}
    -e # forced to change the password at next login
    強迫使用者下次登入時改密碼    		 passwd -e {account}

    密碼設定 (chage)

    另一個有類似功能的指令就是 chage ，如查看使用者密碼狀態

    root:~ # chage-l {account} # 查詢使用者的密碼狀態
    Password: # user須輸入密碼root則不用
    Minimum:        0 # 使用者至少隔幾天才能改密碼
    Maximum:        99999 # 使用者密碼還有幾天會無效
    Warning:        7 # 警告使用者密碼到期的期限
    Inactive:        -1 # 在使用者密碼到期後，所執行的動作
    Last Change:        Jan 28, 2006 # 使用者最後改密碼的日期
    Password Expires:        Never # 使用者密碼到期日
    Password Inactive:        Never #
    Account Expires:        Never # 使用者帳號到期日

    argument

    argument description example
    -m, --mindays MIN # minimum number of days between password changes
    使用者至少幾天隔幾天才能改密碼
    -M, --maxdays MAX # maximum number of days during which a password is valid
    使用者密碼還有幾天會無效
    -W, --warndays WARN # number of days of warning before a password change
    設定當密碼有效期限幾天內，系統會警告使用者密碼無效
    -I, --inactive INACT # number of days of inactivity after a password has expired before the account is locked
    在使用者密碼到期後
    "-1":當使用者下次登入將被強迫更改密碼
    "0":取消這功能，即是沒有什麼事情發生
    -l, --list # list the password expiry information
    查看使用者密碼狀態
    -d, --lastday YYYY-MM-DD # password was last changed can be set to another value
    設定使用者最後修改密碼的時間，以 " 年-月-日" 的形式輸入
    -E, --expiredate YYYY-MM-DD # account will be expired can be changed
    設定使用者帳號到期的時間，以 " 年-月-日 " 的形式輸入

    群組帳號設定

    groupadd

    要增加群組帳號，常使用groupadd這支指令

    root:~ # groupadd {group1} # 增加一個名group1的群組

    arguments

    argument description example
    -g, --gid GID # Force the new group ID to be the given number
    新增群組時並指派 GID    		 groupadd -g 103 {group1}

    groupmod

    要修改群組關係，就要使用groupmod這指令，像是將某個使用者增加到某個群組，或移除等等的關係

    root:~ # groupmod -A {account} {group} # 將使用者account加入到群組 group1

    arguments

    argument description example
    -g, --gid GID # Force the new group ID to be the given number
    修改群組的GID    		 groupmod -g 104 {group1}
    -A, --add-user USER # Add the user account to the specified group
    將使用者加入到某群組，若要同時增加多人可以用 "," 分隔    		 groupmod -A {account} {group1}
    -R, --remove-user USER # Remove the user account from the specified group
    將使用者移從某群組除，若要同時移除多人可以用 "," 分隔    		 groupadd -R {account1}, {account2} {group1}

    groupdel

    要移除群組帳號，常使用 groupdel 這指令，比較少在使用其參數，但是有一點要注意的，就是要移除的群組帳號為某使用者的 primary group （主要群組）時，此群組無法移除，除非此群組帳號之下沒有使用者才行

    root:~ # groupdel {group1} # 移除一個名 group1 的群組

    gpasswd

    設定群組的密碼，用gpasswd(少用)

    root:~ # gpasswd {group1} # 設定群組 group1 的密碼

    argument

    argument description example
    -r # Remove group password gpasswd -r {group1}
