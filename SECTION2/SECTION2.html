    Objective 1: Access and Use man Pages
    user:~ $ man {command} # manual for a command

    手冊欄位說明

    名稱 說明
    NAME 指令名稱
    SYNOPSIS 摘要: 文件使用方式或指令的基本操作語法
    DESCRIPTION 說明: 該指令的完整說明
    OPTION 記錄参數的名稱與用法
    COMMANDS 程式執行時可用的指令
    FILES 與此文件相關的檔案路徑
    SEE ALSO 參考資料路徑
    DIAGNOSTICS 診斷可能之錯誤訊息
    EXAMPLE 語法範例
    BUGS 目前存在尚未解決的bug

    Example:

    user:~ $ man man
    Man: find all matching manual pages
    * man (1)
    man (7)
    man (1p)
    Man: What manual page do you want?
    Man:
    # man這個指令有1, 7, 1p 三個章節

    手冊章節說明

    章節 說明
    0 檔頭文件(Header files)
    1 使用者指令
    2 Linux kernal提供的system call
    3 Lib calls
    4 裝置檔
    5 檔案格式或設定檔案相關語法
    6 遊戲軟體
    7 雜項
    8 管理工具與指令
    9 Linux核心常式(Kernal routines)
    p 程式相關手冊
    # 查詢手冊標題內容 user:~ $ man -f {command}
    user:~ $ whatis {command} # whatis 與-f同樣效果
    user:~ $ man -k {command} # 全文檢索

    Objective 2: Use info Pages

    Info是Linux系統所提供，其文件檔案存放在/usr/share/info下，格式與man則略有不同，他主要是將一份文件分成多個節點，透過超連結的方式讓使用者可以類似瀏覽網頁的方式瀏覽文件內容。

    user:~ $ info {command}

    info操作指令

    指令 說明
    [Enter] 進入
    [tab] 移至下一個超連結
    [alt]+[tab] 移至上一個超連結
    n 移至下一個節點
    p 移至上一個結點
    s、/ (search)搜尋關鍵字
    q quit
